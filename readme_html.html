Описание примененных техник, инструментов и тест-кейсов для тестирования форм 'Регистрация' и 'Авторизация' на сайте 'Ростелеком'

Для тестирования форм "Регистрация" и "Авторизация" на сайте сайте "Ростелеком" были применены следующие техники:

  • Ручное функциональное тестирование
  • Автоматизированное тестирование

Тест-кейсы, разработанные для этого проекта в основном на требованиях брифа предоставленного заказчиком, предназначены для ручного тестирования и описывают шаги, которые тестировщик должен выполнить для проверки определенных аспектов функциональности сайта:

  • Вход через социальные сети
  • Проверка ссылок пользовательского соглашения
  • Ввод некорректных данных
  • Другие сценарии

Для описания баг-репортов использовалась Google Таблица, которая позволяет структурировать и отслеживать информацию о найденных проблемах. В таблице указывается описание бага, шаги для воспроизведения, ожидаемое поведение и фактический результат, а также прикрепляются дополнительные материалы, такие как скриншоты или логи.

Использование комбинации функционального и автоматизированного тестирования позволяет эффективно проверить функциональность сайта "Ростелеком", выявить потенциальные проблемы и автоматизировать процесс тестирования для повторного использования в будущем. Для написания и запуска тестов был выбран фреймворк PyTest, который обладает широким набором функциональности и простым синтаксисом. Взаимодействие с веб-страницами и выполнение действий, таких как ввод данных и проверка результатов, осуществлялось с помощью Selenium WebDriver.

Каждый тест проверял конкретные аспекты функциональности страниц форм регистрации и авторизации, включая:

  • Регистрацию по емайлу
  • Авторизацию по номеру телефона
  • Работу элементов страницы, таких как ссылки пользовательского соглашения, чек-бокс "Запомнить меня" и поля ввода данных
  • Проверку сообщений об ошибках при вводе неверных данных
  • Другие аспекты

Автотесты, проведенные для форм "Регистрация" и "Авторизация" на сайте "Ростелеком", относятся к категории UI (User Interface) тестов. Они проверяют функциональность и взаимодействие пользовательского интерфейса данных форм, включая отображение элементов на странице, корректность работы полей ввода данных, функциональность кнопок и других элементов интерфейса. Также UI тесты могут включать проверку сообщений об ошибках и других элементов, связанных с пользовательским опытом.

В целом, использование PyTest и Selenium WebDriver позволило автоматизировать тестирование форм "Регистрация" и "Авторизация" на сайте "Ростелеком", обеспечить проверку функциональности и убедиться, что различные элементы и действия работают корректно.
